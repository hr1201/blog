# 打包TS项目


Rollup打包TS项目

实现了Rollup识别ts，代码压缩，开启端口服务，热更新，以及在rollup.config.js用replace配置环境变量，还可以实现本地生成环境分开，在index.html右键点击Open with Live Server开启生产环境，终端npm run serve开启开发环境。在rollup.config.js中的terser中使用compress删除console。

### 安装依赖

1. 全局安装rollup npm install rollup-g
2. 安装TypeScript   npm install typescript -D
3. 安装TypeScript 转换器 npm install rollup-plugin-typescript2 -D
4. 安装代码压缩插件 npm install rollup-plugin-terser -D
5. 安装rollupweb服务 npm install rollup-plugin-serve -D
6. 安装热更新 npm install rollup-plugin-livereload -D
7. 引入外部依赖 npm install rollup-plugin-node-resolve -D
8. 安装配置环境变量用来区分本地和生产，在package.json中的dev(开发环境)和build(生产环境)使用 npm install cross-env -D
9. 替换环境变量给浏览器使用 npm install rollup-plugin-replace -D

## rollup.config.js

```typescript
// 用于路径
import path from 'path'
// 使rollup可以识别到ts
import ts from 'rollup-plugin-typescript2'
// 开启端口服务
import serve from 'rollup-plugin-serve'
// 热更新，也就是保存自动刷新页面
import livereload from 'rollup-plugin-livereload'
// 用于代码压缩
import { terser } from 'rollup-plugin-terser'
// 用于在浏览器配置一个process
import replace from 'rollup-plugin-replace'

// 用于判断process.env.NODE_ENV是否为开发环境
const isDev = () => {
    return process.env.NODE_ENV === 'development'
}

export default {
    // 入口
    input: './src/index.ts',
    // 出口
    output: {
        // 文件位置
        file: path.resolve(__dirname, './lib/index.js'),
        // 输出格式
        format: 'umd',
        sourcemap: true//使代码规范化
    },

    plugins: [
        ts(),
        // 是开发环境时才开启热更新
        isDev() && livereload(),
        terser({
            compress:{
                // 可以用于删除console
                drop_console:true
            }
        }),
        replace({
            // 把process.env.NODE_ENV注册到浏览器里
            'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV)
        }),
        // 是开发环境时才开启端口
        isDev() && serve({
            open: true,
            port: 1996,
            openPage: '/public/index.html'
        })
    ]
}
```
